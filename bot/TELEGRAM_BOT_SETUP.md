# 🤖 Настройка Telegram бота WeatherNews

## ✅ Что уже сделано:

### 1. База данных переведена на SQLite
- Изменена конфигурация с PostgreSQL на SQLite
- Создан файл `bot.db` в корне папки bot
- Добавлены все таблицы согласно ТЗ:
  - `users` - пользователи
  - `favorite_cities` - избранные города
  - `news_categories` - категории новостей
  - `user_subscriptions` - подписки на категории
  - `weather_history` - история запросов погоды
  - `weather_cache` - кэш погоды
  - `news_cache` - кэш новостей
  - `activity_logs` - логи активности

### 2. Модели данных обновлены
- Все модели соответствуют ТЗ
- Добавлены связи между таблицами
- Добавлены индексы для оптимизации

### 3. Базовые команды работают
- `/start` - регистрация и приветствие
- `/help` - справка по командам
- `/menu` - главное меню
- `/about` - информация о боте
- `/weather [город]` - текущая погода
- `/forecast [город] [дни]` - прогноз
- `/mycity` - погода в сохраненном городе

- `/feedback [текст]` - отзыв

## 🚀 Быстрый запуск:

### 1. Инициализация базы данных:

```bash
cd bot
source venv/bin/activate
python init_db.py
```

Это создаст все таблицы и заполнит категории новостей.

### 2. Проверка конфигурации:

Убедитесь, что в `bot/.env` есть:
```env
BOT_TOKEN=ваш_токен_бота
WEATHER_API_KEY=ваш_ключ_погоды
NEWS_API_KEY=ваш_ключ_новостей
```

### 3. Запуск бота:

```bash
# Из корня проекта
./start.sh

# Или только бот
cd bot
python main.py
```

## 🐛 Известные проблемы:

1. **Redis не запущен** - бот требует Redis для работы
   - Решение: `sudo apt install redis-server && sudo systemctl start redis`
   - Или закомментировать Redis в `main.py` для тестирования

2. **NEWS_API_KEY неверный** - возвращает 401
   - Решение: Получить новый ключ на https://newsapi.org
   - Или бот будет использовать mock данные

3. **Некоторые команды не реализованы**
   - См. список выше в разделе "Что нужно доработать"
