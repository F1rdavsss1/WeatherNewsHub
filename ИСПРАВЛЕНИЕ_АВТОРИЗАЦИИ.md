# ✅ Исправление авторизации через Telegram

## Проблема
Коды авторизации, генерируемые ботом, не работали на сайте. Причина: бот и API сервер работали в разных процессах Python и не разделяли общую память.

## Решение
Изменено хранилище кодов авторизации с in-memory словаря на SQLite базу данных (`bot.db`), которая доступна обоим процессам.

## Что было изменено

### 1. `bot/app/auth_storage.py`
- Заменено хранилище с `dict` на SQLite
- Добавлена таблица `auth_codes` в базе данных
- Коды теперь сохраняются в БД и доступны всем процессам

### 2. `bot/app/config.py`
- Сделаны необязательными поля `WEATHER_API_KEY` и `NEWS_API_KEY` (для тестирования)

## Как запустить

### Вариант 1: Автоматический запуск (рекомендуется)
```bash
./start.sh
```

### Вариант 2: Ручной запуск

1. Запустите API сервер:
```bash
cd bot
source venv/bin/activate
python3 api_server.py
```

2. В новом терминале запустите бота:
```bash
cd bot
source venv/bin/activate
python3 main.py
```

3. В третьем терминале запустите фронтенд:
```bash
cd front
npm run dev
```

## Как проверить

1. Откройте Telegram бот: [@WeatherNewsHubot](https://t.me/WeatherNewsHubot)
2. Отправьте команду `/login` или нажмите кнопку "🔐 Вход на сайт"
3. Скопируйте полученный 6-значный код

## Технические детали

### Структура таблицы auth_codes
```sql
CREATE TABLE auth_codes (
    code TEXT PRIMARY KEY,
    user_data TEXT NOT NULL,
    expires_at TEXT NOT NULL,
    created_at TEXT DEFAULT CURRENT_TIMESTAMP
)
```

### Процесс авторизации
1. Пользователь отправляет `/login` в Telegram
2. Бот генерирует 6-значный код
3. Бот сохраняет код в `bot.db` (срок действия: 5 минут)
4. Пользователь вводит код на сайте
5. Фронтенд отправляет код на API (`POST /auth/telegram`)
6. API проверяет код в `bot.db`
7. Если код валиден, API возвращает данные пользователя
8. Код удаляется из БД (одноразовый)

## Логи

## Возможные проблемы

### Бот не запускается
- Проверьте `BOT_TOKEN` в `bot/.env`
- Убедитесь, что токен валидный

### API не отвечает
- Проверьте, что порт 8000 свободен: `lsof -i :8000`
- Проверьте логи: `tail -f logs/api.log`

### Код не работает
- Убедитесь, что оба сервиса (бот и API) запущены
- Проверьте, что используется одна и та же база данных `bot/bot.db`
- Код действителен только 5 минут
- Код одноразовый (после использования удаляется)
